0001-01-01 ALG-Segmentation

Category

  語意的部份就是全部的氣球,都是同一種     
</td>

  => 找到7個objects
</td>

  => instance segmentation,和object detection差異是 除了找到7個objects,
  還有找到這些objects對應的pixels(MaskRCNN就是屬於這一種)
</td>

Comparison

  Encoder交替采用
  (1)conv + (2)pooling，

  Decoder交替采用 
  (1)deconv+ (2)upsampling，用 (3)Softmax做像素分类。
  
  在Encoder-Decoder过程中，
  采用Pooling Indices（pooling时的位置信息）转移 Decoder 
</td>

  U-Net没有利用池化位置索引信息，
  而是将编码阶段的整个特征图传输到相应的解码器
  （以牺牲更多内存为代价）
  => short-cut 
</td>

  然後再透過CFF(feature fusion)將特徵合為一起,然後再做upsample,
  即可變回原圖大小 with semantic segmentation
  比SegNet 大大地提高了准确率，足以与Deeplab v2媲美
</td>

  encode端：       
  deeplabv3用"原圖"搭配不同的dilation conv來擷取特徵 
  => 所以圖不會變小  
  
  encode端： 
  U-Net透過不同的conv,不斷做down sample, 
  得到最後的encode結果,但decode的時候,會用之前的資料
</td>

  deeplabv3
  是直接upsample回目標大小
  (DeepLabv3直接用encode到最後的值
  (不同dilation conv)直接upsample
  
  deeplabv3+ 
  => 現在則是分兩段upsample，第一段upsample完之後，
  => 與encoder中feature map大小一樣的low level feature組合起來
  (所以會做個merge)，  
  => 再upsample到目標大小
  
  (v3+ decode則是會和1x1 conv組合"和feature map一樣大小")
  => 所以v3+是加強了decode的部份
</td>

  在实现目标检测的同时，把目标像素分割出来 
  =>所以可以比SSD這種類型,再多做一點 
  
  和Faster - RCNN 差異 
  1）将 Roi Pooling 层替换成了 RoiAlign；
  2）添加并列的 FCN 层（mask 层）；=>所以叫作"Mask"RCNN
    FPN 在 MaskRCNN.build() 中创建，位于构建 ResNet 的部分之后 
    (Mask R-CNN 的同一作者引入的特征金字塔网络（FPN）是对该主干网络的扩展，
    可以在多个尺度上更好地表征目标) 
</td>

可以看到下圖 object detection, 只能框住框匡
deeplabv3 (semantic segemetation),只能切割類別
maskrcnn,可以裝住框框(分類),又可以做切割 (instance segmentation)
