0001-01-01 SW-OP-Depthwise-conv

Traditional CONV

舉個例子
input = [224,224,3,1] RGB的圖
output = [224,224,10,1] 變成有10個channel
這樣的話kernel 可以是[5,5,3,10]

如下圖(1), input channel 一定要等於 kernel channel
如下圖(2), kernel個數 一定要等於 output channel
如下圖(3), output每個點的計算等於 input channel 和對應的 filter channel做計算再加起來
以範例來看就是
input channel 1 <=> kernel 1 + input channel 2 <=> kernel 2 + input channel 3 <=> kernel 3

W_in = 224
H_in = 224 Nch = 3
k = 5
Nk = 10
可以這樣想,output = [224,224,10,1]

每一個output的點 都要做 k * k * Nch 運算

(所以多個channel的資料 都會被參考到, 所以這也是為什麼conv會比depthwise conv來的準一點)

總共ouput點有這些 W_out * H_out * Nk

一般卷積 計算量為 = W_out * H_out * Nk * k * k * Nch

也可以由這個例子看出,input channel 和對應的 filter channel做計算再加起來的感覺

所以也可以看到input中的channel和filter中的channel一定是一對一的關係,e.g. channel 1對channel 1,

但output channel不只有一,所以filter才會有很多組(batch=>dimension 3)

Depthwise CONV

step 1: Depthwise convolution
針對輸入資料的每一個Channel都建立一個k*k的Kernel，然後每一個Channel針對對應的Kernel都各自(分開)做convolution。
這步驟和一般卷積不太一樣，一般的卷積計算是每個Kernel Map都要和所有channel都去做convolution，

這邊是分開獨立去做

step 2: Pointwise convolution
在輸入資料的每個channel做完depthwise convolution後，針對每個點的所有channel做pointwise convolution。
實際做法是說建立Nk個11Nch的kernel Map，將depthwise convolution的輸出做一般1*1的卷積計算

Pointwise Convolution的运算与常规卷积运算非常相似，
它的卷积核的尺寸为 1×1×Nch，Nch为上一层的通道数。

所以这里的卷积运算会将上一步的map在深度方向上进行加权组合，生成新的Feature map。

有几个卷积核就有几个输出Feature map (幾組filter,Nk,output channel就會有多少)

所以有個結論是不同的filter,彼此都是沒有關係的,不論是CONV,depthwise conv, pointwise conv

Depthwise separable convolution計算量為
(Depthwise convolution計算量+Pointwise convolution計算量)
Depthwise convolution 完後會有 W_out * H_out * Nch 個點, 每個點只要做 k * k 即可

所以 Depthwise convolution 計算量為 W_out * H_out * Nch* k * k

Pointwise convolution 完後會有 Nk * W_out * H_out, 每個點要做 Nch的相加(如下第二個圖)

所以 Pointwise convolution 計算量為 Nch * Nk * W_out * H_out

Depthwise conv (這個產生的channel維度,和input相同)

Pointwise conv (可以透過有多組的filter,來調整output channel維度)

運算量比較 (下圖不是Win是Wout)

=> 所以可以看到在pointwise conv, k的因素已經不見了,這個就是減少計算量的原因

=> 因為pointwise conv也是傳統的conv, 原本每個output點的計算量是 k * k * Nch (input有幾個channel)

=> 但pointwise conv是k=1 => 所以每個output點的計算量就變成 Nch (input有幾個channel)

=> 因此如果k 愈大的case(因為pointwise沒有理會k), and Nk 不要太小, depthwise+pointwise, 就會很有用
