0001-01-01 SW-FRAMEWORK-Xiaomi-Mace

Concept

Mobile AI Compute Engine (MACE)
是一个专为移动端异构计算设备优化的深度学习前向预测框架。
=> 所以主要就是做inference
MACE覆盖了常见的移动端计算设备（CPU、GPU、Hexagon DSP、Hexagon HTA、MTK APU），
并且提供了完整的工具链和文档，用户借助MACE能够 很方便地在移动端部署深度学习模型。
MACE已经在小米内部广泛使用并且被充分验证具有业界领先的性能和稳定性

Prepare 0 (python)

conda activate mace
還有要安裝mace要的requirement

Prepare 1 (bazel)

要安裝bazel 0.13 (原本是0.26)
注意把bazel用user版本安裝,這樣在build mace的時候,才可以不用root build,用root會有問題
=> sudo ./bazel-0.13.1-installer-linux-x86_64.sh –user –bin=/home/evan/bin-user

export BAZEL_VERSION=0.13.1
mkdir bazel-user && \
    cd bazel-user && \
    wget https://github.com/bazelbuild/bazel/releases/download/
    $BAZEL_VERSION/bazel-$BAZEL_VERSION-installer-linux-x86_64.sh && \
    chmod +x bazel-*.sh && \
    sudo ./bazel-0.13.1-installer-linux-x86_64.sh --user --bin=/home/evan/bin-user && \
    cd / && \
    rm -f bazel-user/bazel-$BAZEL_VERSION-installer-linux-x86_64.sh

然後要把tools/bazel_build_standalone_lib.sh裡面的
bazel都改用自己的user build版本
/home/evan/bin-user/bazel
如果不這樣做的話,在build mace的時候會出現
Issue: cp: failed to preserve ownership

Prepare 2 (NDK)

不能用太新的會有問題
The major revision of the Android NDK referenced by android_ndk_repository rule ‘androidndk’ is 18.
The major revisions supported by Bazel are [10, 11, 12, 13, 14, 15, 16].
Bazel will attempt to treat the NDK as if it was r16. This may cause compilation and linkage problems

wget -q https://dl.google.com/android/repository/android-ndk-r15c-linux-x86_64.zip && \
unzip -q android-ndk-r15c-linux-x86_64.zip && \
rm -f android-ndk-r15c-linux-x86_64.zip

然後還要設定Android NDK
要修改目錄底下的WORKSPACE,這個和tensorflow一樣

android_ndk_repository(
    name = "androidndk",
    path="/home/evan/SDK/mace/android-ndk-r15c",  
    api_level=21,
)

Build

runtime可以指定要build那些backend
default只有CPU,但可以多加GPU and DSP
然後預設都是build成static library
bash tools/bazel_build_standalone_lib.sh -abi=arm64-v8a -runtimes=CPU,GPU -static
注意不要下sudo bash => 因為他只能支援user build
