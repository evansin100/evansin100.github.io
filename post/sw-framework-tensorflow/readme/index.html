0001-01-01 SW-FRAMEWORK-Tensorflow

Tensorflow build from source

git clone https://github.com/tensorflow/tensorflow.git

git branch -a
然後選擇要的版本

cd tensorflow # cd to the top-level directory created
=> ./configure
因為build這個會需要bazel但是tensorflow會指定　　 bazel的版本
解決的方是install from script
先去官網
(使用舊的版本比較不會有問題）
For TF1.12, you need to downgrade Bazel to 0.15.0.
For TF1.13.1, you need to use Bazel 0.19.2. Thanks!
https://github.com/bazelbuild/bazel/releases/tag/0.27.1
下載對應的版本
chmod +x bazel--installer-linux-x86_64.sh
./bazel--installer-linux-x86_64.sh –user
The –user flag installs Bazel to the $HOME/bin directory on your system
所以會被安裝在home/bin
然後再指定目錄即可
export PATH="$PATH:$HOME/bin”
如果之前有安裝過bazel可以透過下面指令把它砍掉
sudo rm -rf /usr/bin/bazel
可以利用bazel which來找到binary的位置

CPU-only support
To build a pip package for TensorFlow with CPU-only support:
=> bazel build –config=opt //tensorflow/tools/pip_package:build_pip_package

產生安裝檔
bazel-bin/tensorflow/tools/pip_package/build_pip_package /tmp/tensorflow_pkg

安裝tensorflow
sudo pip uninstall tensorflow => 可以先unintall以免有些檔案干擾
sudo pip install /tmp/tensorflow_pkg/tensorflow-1.xxxx

安裝完了之後,可以再不同的路徑,不要在tensorflow的root folder
打下以下的指令,來確認安裝的版本是否正確
python -c ‘import tensorflow as tf; print(tf.version)’
